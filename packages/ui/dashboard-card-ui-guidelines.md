# Dashboard Card UI Guidelines

## 목표 인상

- 가족 앨범과 생활 허브의 중간 지점을 만든다.
- 차갑거나 업무용 대시보드 같은 인상은 피한다.
- 그렇다고 지나치게 장식적이지 않게, 한 번에 읽히는 정보 밀도를 유지한다.

## 카드 계층

| 크기 | 권장 용도 | 예시 |
| --- | --- | --- |
| `XL` | 첫 화면 핵심 메시지 | Hero, 긴급 공지 |
| `L` | 오늘 행동 유도 카드 | 오늘 일정, 오늘 할 일, 중요 공지 |
| `M` | 상태 요약 카드 | 진행률, 루틴, 최근 글 |
| `S` | 보조 카드 | 최근 사진, D-day, 고정 메모 |

## 형태 가이드

- 모서리: 둥근 카드 기반
- 표면: 완전 흰색보다 크림 또는 옅은 샌드 톤 우선
- 그림자: 깊은 그림자보다 부드러운 레이어 분리 중심
- 간격: 정보가 많은 카드일수록 내부 여백을 넉넉히 둔다

## 정보 우선순위

1. 카드 제목
2. 짧은 상태 라벨 또는 배지
3. 한 줄 요약
4. 날짜/시간 또는 수치
5. 보조 액션

## 배지 규칙

- `긴급`: 가장 강한 대비색
- `중요`: 명확한 강조색
- `오늘`: 따뜻한 포인트색
- `지연`: 경고 톤
- `D-day`: 감성 카드에서도 읽히는 선명한 강조

## 섹션별 카드 표현

### Hero

- 사진 또는 부드러운 패턴 배경을 허용한다
- 가족 이름, 날짜, 대표 메시지, 최우선 카드 한 개를 함께 보여준다
- 텍스트가 사진에 묻히지 않게 오버레이 대비를 확보한다

### Today

- 일정과 투두는 한 화면에서 바로 비교되게 맞춘다
- 시간, 마감, 완료 개수는 숫자가 바로 보이게 한다

### Focus

- 중요 공지, 읽음 확인, 지연된 항목 같은 강한 행동 유도 카드용
- 배지와 제목만 봐도 이유를 이해할 수 있어야 한다

### Progress

- 원형 차트보다 퍼센트, 진행 바, 연속 일수처럼 즉시 읽히는 표현을 우선한다
- 가족 공통 목표와 개인 루틴은 같은 비주얼 문법을 사용하되 라벨로 구분한다

### Recent

- 글은 텍스트 미리보기 중심
- 사진은 썸네일 중심
- 두 카드가 나란히 와도 톤이 어색하지 않도록 동일한 카드 프레임을 쓴다

### Pinned

- 기념일, 메모, 가족 공지 같은 장기 카드
- 너무 커지지 않게 `S` 또는 `M` 크기를 기본으로 한다

## 상태 표현

- `featured`: 카드 상단 강조선 또는 배경 틴트
- `pinned`: 핀 아이콘보다 "고정" 성격이 드러나는 라벨 우선
- `read receipt pending`: 확인 필요 라벨과 액션 버튼 분리
- `overdue`: 날짜만 붉게 처리하지 말고 전체 위험도를 함께 보여준다

## 분위기 초안

- 기본 팔레트: 크림, 아이보리, 샌드, 세이지
- 포인트: 코랄 또는 딥 그린 중 하나를 강조색 후보로 사용
- 배경: 단색 흰색보다 옅은 그라데이션이나 종이 질감 느낌이 어울린다

## 모바일 우선 규칙

- 카드 하나의 목적이 두 개를 넘지 않게 한다
- 접기 전 화면에서 핵심 3카드가 자연스럽게 이어져야 한다
- 최근 글/사진은 과도하게 길어지지 않도록 요약 길이를 제한한다

## 아직 열어둘 항목

- 가족 사진을 Hero에 항상 둘지, 테마 설정값으로 뺄지
- 진행률 카드에 차트를 허용할지
- `child` 사용자에게 보여줄 색 대비와 정보 밀도를 별도로 둘지
