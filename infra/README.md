# infra

배포, 환경설정, 운영 문서, Docker 또는 호스팅 관련 자산을 두는 공간입니다.

## 내부망 운영 기준

- 대표 접속 경로를 하나로 고정합니다.
- 개발/운영 LAN 실행은 `npm run dev:lan`, `npm run start:lan` 기준으로 맞춥니다.
- 기본 점검 경로는 `/api/health`, `/`, `/console`, `/f/{slug}`, `/app/{slug}`입니다.
- 가족 입구와 운영 콘솔은 같은 서버를 쓰더라도 동선을 분리해 운영합니다.
- 파일 저장을 유지하는 동안 `apps/web/data/family-sites.json` 백업 주기와 복구 절차를 문서화합니다.

## 파일 저장 -> DB 저장 준비 체크리스트

- 현재 파일 저장 필드와 Prisma 스키마 필드를 1:1로 매핑합니다.
- `accessSecret` 평문 저장을 해시 또는 별도 비밀 저장소 방식으로 분리합니다.
- 파일/DB 이중 읽기 기간의 우선순위를 정합니다.
- 기존 `family-sites.json`을 적재하는 백필 스크립트와 롤백 절차를 준비합니다.
- 저장 전환 후에도 가족 생성, 빌더 저장, 리셋, 내부망 조회 결과가 같도록 회귀 테스트를 준비합니다.

## 배포 준비 체크리스트

- 환경변수 계약: LAN 전용 실행과 배포 실행이 어떤 값으로 갈리는지 정리
- 저장 백엔드 스위치: 파일 저장인지 DB 저장인지 명시적 분기 준비
- 기본 도메인/커스텀 도메인 라우팅 기준 준비
- `/api/health` 기반 헬스체크와 배포 후 스모크 테스트 묶음 준비
- 데이터 백업과 롤백 순서 문서화
