# 가족 플랫폼 초안

이 문서는 현재 기준의 실무형 초안입니다. 구현 순서, 모듈 범위, 공지 양식, 확장 전략을 한 번에 보는 용도로 씁니다.

## 1. 제품 한 줄 정의

가족별로 커스터마이징 가능한 메인 홈과 여러 생활형 게시판을 제공하는 멀티테넌트 가족 플랫폼.

## 2. 지금 당장 고정할 원칙

- 처음부터 모든 기능을 완성하지 않는다.
- 대신 새 모듈을 쉽게 붙일 수 있는 공통 구조를 먼저 만든다.
- 메인 홈은 모든 모듈이 연결되는 허브로 설계한다.
- 공지 게시판은 초기에 반드시 넣는다.
- 진행률/습관 카드는 플랫폼의 차별화 포인트로 본다.

## 3. 게시판/모듈 분류 초안

### 콘텐츠형

- 공지 게시판
- 일반 글 게시판
- 일기
- 가족 규칙/가이드

### 미디어형

- 갤러리
- 추억 보관함
- 파일 보관함

### 체크형

- 투두 리스트
- 장보기 리스트
- 습관 체크

### 시간형

- 일정/캘린더
- 학교/학원 시간표
- 24시간 하루 시간표

### 트래커형

- 진행률 보드
- 목표 관리
- 공부/운동/저축 진행률
- 연속 기록 보드

## 4. MVP 우선순위 초안

### MVP 1차

- 첫 화면 스플래시
- `들어가기` 버튼과 가족 비밀번호 진입
- 메인 홈
- 공지 게시판
- 일반 글 게시판
- 일정/캘린더
- 투두 리스트

이 단계만 완료되어도 플랫폼의 기본 뼈대와 실제 사용감이 생깁니다.

### MVP 2차

- 갤러리
- 일기
- 학교/학원 시간표
- 24시간 하루 시간표
- 진행률 보드
- 습관 체크

이 단계부터 "가족 생활 플랫폼"다운 느낌이 강해집니다.

### 확장 단계

- 장보기 리스트
- 식단표
- 기념일/D-day
- 추억 보관함
- 설문/투표
- 파일 보관함
- 가계부 라이트

## 5. 공지 게시판 양식 초안

공지 게시판은 일반 글 게시판과 다르게 "전달"과 "확인"이 핵심이라 별도 양식이 필요합니다.

기본 필드:

- 제목
- 공지 유형: 일반 / 중요 / 긴급
- 상단 고정 여부
- 메인 노출 여부
- 메인 우선순위
- 노출 시작일 / 종료일
- 대상 범위
- 본문
- 첨부파일
- 댓글 허용 여부
- 읽음 확인 여부

리스트 화면에서 바로 보여야 할 정보:

- 중요도 배지
- 고정 여부
- 노출 기간
- 읽음 여부

## 6. 진행률/습관 보드 초안

### 진행률 보드 타입

- 퍼센트형: 0~100%
- 개수형: 완료 수 / 목표 수
- 단계형: 준비중 / 진행중 / 완료
- 연속형: 며칠 연속 달성

### 활용 예시

- 시험공부 진도율
- 청소 루틴 완료율
- 운동 연속 기록
- 여행 준비 체크율
- 저축 목표 달성률

## 7. 메인 홈 노출 규칙 초안

모든 모듈은 아래 체크 규칙으로 메인에 노출됩니다.

- `메인에 보이기`
- `고정`
- `우선순위`
- `노출 시작일`
- `노출 종료일`

메인 홈 기본 섹션:

- 오늘 할 일
- 오늘 일정
- 중요 공지
- 최근 글/사진
- 진행률/습관 카드
- 기념일/D-day

- 공지
- 일반 글

- 일기

### 스트림 E. 시간형/체크형 모듈

- 일정

- 하루 시간표

- 배포

## 9. 결론

지금은 "다 만들고 시작"보다 "핵심 구조를 만들고 계속 붙이는 방식"이 맞습니다.

권장 순서는 다음과 같습니다.

1. 진입 화면 + 메인 홈 + 공지
2. 일반 글 + 일정 + 투두
3. 갤러리 + 시간표 + 진행률
4. 플랫폼 설정 + 배포
