# 구현 로드맵

## Phase 0. 기획 고정

목표는 "한 가족용 사이트"가 아니라 "가족 플랫폼"이라는 점을 팀 안에서 고정하는 것입니다.

완료 기준:

- 제품 범위 문서 정리
- 폴더 구조 확정
- 병렬 작업 스트림 정의
- 기본 기술 방향 합의
- 모듈 타입 체계와 MVP 범위 고정

## Phase 1. 기반 공사

가장 먼저 해야 할 일은 예쁜 화면이 아니라 흔들리지 않는 바닥입니다.

범위:

- monorepo 초기화
- 공용 설정 정리
- 앱 기본 라우팅
- 디자인 토큰
- DB 연결
- 기본 인증 뼈대
- 테넌트 식별 구조
- 공통 게시판 메타데이터 계약

완료 기준:

- 빈 화면이어도 가족 slug 단위로 진입 가능
- 개발/빌드/테스트 명령 체계 정리
- DB 스키마 첫 버전 작성

## Phase 2. 진입 경험 + 메인 홈

사용자가 처음 "이 서비스 좋다"를 느끼는 구간입니다.

범위:

- 첫 스플래시 화면
- `들어가기` 버튼
- 비밀번호/접근 코드 입력
- 가족 홈 레이아웃
- 메인 카드/위젯 기본 구성
- 공지/일정/투두/진행률 카드 슬롯 정의

완료 기준:

- 가족 홈으로 실제 진입 가능
- 메인에 샘플 카드 노출 가능
- 모바일과 데스크톱 모두 자연스럽게 보임

## Phase 3. 핵심 모듈 MVP

먼저 자주 쓰이는 기능부터 안정적으로 만듭니다.

범위:

- 공지 게시판
- 일반 게시판
- 갤러리
- 투두
- 일정/캘린더
- 공지 게시판 기본 양식
- 메인 노출 체크와 고정 기능

완료 기준:

- 각 모듈에서 생성/수정/삭제 가능
- `메인에 보이기` 체크 가능
- 메인 홈에서 모듈 데이터를 집계 가능
- 공지 읽음 확인과 노출 기간이 동작함

## Phase 4. 생활형 모듈 확장

가족 플랫폼다운 사용감을 만드는 보조 모듈들을 추가하는 단계입니다.

범위:

- 일기
- 학교/학원 시간표
- 24시간 하루 시간표
- 진행률 보드
- 습관 체크

완료 기준:

- 개인형/가족형 시간 관리가 가능함
- 진행률과 연속 기록이 메인 홈 카드로 보임
- 루틴 데이터가 누적되기 시작함

## Phase 5. 플랫폼화

이 단계부터 "우리 가족만 쓰는 앱"이 아니라 "다른 가족도 쓰는 플랫폼" 느낌이 살아납니다.

범위:

- 가족 생성 플로우
- 테마 변경
- 메뉴 구성
- 활성 모듈 선택
- 권한 정책
- 관리자 설정 화면

완료 기준:

- 가족별 설정이 완전히 분리됨
- 가족 A/B의 홈 구성이 달라짐
- 모듈 on/off가 동작함

## Phase 6. 배포와 운영

배포가 단순히 되는 수준이 아니라, 나중에 유지보수 가능한 상태까지 가져갑니다.

범위:

- 프로덕션 배포
- 환경변수 설계
- 이미지 저장소 연결
- 로그/모니터링
- 백업 정책
- 커스텀 도메인 검토

완료 기준:

- 실제 외부 접속 가능
- 장애 확인 포인트 확보
- 최소한의 운영 문서 확보

## Phase 7. 확장 모듈

초기 안정화 후 아래 모듈을 순차 추가합니다.

- 장보기 리스트
- 식단표
- 추억 보관함
- 설문/투표
- 가족 규칙
- 습관 추적
- 파일 보관함
- 가계부 라이트
