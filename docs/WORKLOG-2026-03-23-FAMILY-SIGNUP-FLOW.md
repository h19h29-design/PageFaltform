# YSplan 작업기록 - 2026-03-23 가족 가입 흐름

## 목표

- 관리자가 새 가족 홈 하나를 만든다.
- 비회원 사용자가 그 가족 링크로 들어간다.
- 비회원이 회원가입 후 그 가족에 연결된다.
- 가족 입장 비밀값을 넣고 실제 앱을 사용한다.

## 이번 작업

- 가족 입구에서 바로 `회원가입` 과 `로그인` 으로 갈 수 있게 연결했습니다.
- `회원가입` 링크는 현재 가족 슬러그를 함께 넘겨서, 가입 화면에서 그 가족이 기본 선택되도록 만들었습니다.
- 가입이 끝나면 같은 가족의 입장 단계로 돌아오도록 유지했습니다.
- 가족/플랫폼 오류 메시지를 한글로 바꿨습니다.
- 새 가족 홈은 공개 가족 목록에 포함되고, 게시판/일정/트래커 데이터는 첫 접근 시 기본 샘플이 자동으로 준비됩니다.

- `C:/gpt/01project/YSplan/apps/web/app/(family)/f/[familySlug]/page.tsx`
- `C:/gpt/01project/YSplan/apps/web/app/(public)/sign-up/page.tsx`
- `C:/gpt/01project/YSplan/apps/web/src/lib/messages.ts`

## 테스트 순서

1. 관리자 로그인  
   `http://127.0.0.1:3001/console/sign-in`

2. 새 가족 홈 만들기  
   `http://127.0.0.1:3001/console/families/new`

3. 생성 후 가족 입구 주소 확인  
   예: `http://127.0.0.1:3001/f/sample-one`

4. 다른 브라우저 또는 비회원 상태에서 그 가족 입구 열기

5. 가족 입구에서 `회원가입` 버튼 클릭

6. 가입 화면에서 가족이 이미 선택돼 있는지 확인

7. 가입 완료 후 같은 가족 입장 단계로 이동하는지 확인

8. 가족 비밀값을 입력하고 `/app/[familySlug]` 로 들어가 실제 사용

## 확인 포인트

- 가족 입구에 `회원가입` 버튼이 보이는지
- 가입 화면에서 선택 가족이 기본값으로 잡히는지
- 가입 후 가족 입장 단계로 다시 돌아오는지
- 입장 후 게시판/일정/체크리스트/시간표/목표가 비어 있지 않고 샘플 데이터가 보이는지
