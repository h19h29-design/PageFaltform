# B-page 역할별 UI 재정리

작성일: 2026-03-31

## 이번에 바꾼 것

- 공개 화면은 `선택과 진입` 중심으로 줄였습니다.
- 회원 화면은 `실제 사용` 중심으로 바꿨습니다.
- 관리자 화면은 `백스테이지`처럼 뒤쪽에 두고, 승인과 설정만 먼저 보이게 바꿨습니다.
- 가족 공개 목록에서는 앱 직링크보다 `입구` 중심으로 바꿔 실제 사용 흐름과 맞췄습니다.

## 역할별 화면 기준

### 공개 화면

- 대상: `/`, `/family`, `/club`, `/clubs`, `/clubs/[clubSlug]`, `/clubs/[clubSlug]/join`, `/sign-in`, `/sign-up`, `/f/[familySlug]`
- 목적: 소개보다 빠른 진입
- 변화:
  - 긴 설명 축소
  - CTA 우선
  - `백스테이지`는 보조 링크로만 노출

### 회원 화면

- 대상: `/app/[familySlug]`, 가족 모듈 상세, `/clubs/[clubSlug]/app`, `/clubs/[clubSlug]/app/[moduleKey]`
- 목적: 실제 사용
- 변화:
  - 홈/게시판/모듈이 먼저 보이게 조정
  - 상태 요약은 얇게
  - 관리 링크는 하단 `백스테이지`로 이동

### 관리자 화면

- 대상: `/console`, `/console/sign-in`, `/console/families/*`, `/console/clubs/*`, `/console/themes`
- 목적: 운영 설정과 승인 처리
- 변화:
  - 제목/부제 축소
  - 생성 버튼은 `운영 도구`로 축소
  - 승인 대기와 설정이 먼저 보이게 정리

## 바로 테스트할 주소

- 메인: `http://127.0.0.1:3001/`
- 가족 공개: `http://127.0.0.1:3001/family`
- 클럽 공개: `http://127.0.0.1:3001/club`
- 가족 입구: `http://127.0.0.1:3001/f/yoon`
- 회원 로그인: `http://127.0.0.1:3001/sign-in`
- 회원가입: `http://127.0.0.1:3001/sign-up`
- 콘솔 로그인: `http://127.0.0.1:3001/console/sign-in`
- 콘솔: `http://127.0.0.1:3001/console`

## 현재 미구현 또는 부분 구현

### 클럽 멤버 앱

- 현재는 `읽기 중심`입니다.
- 되는 것:
  - 공개 탐색
  - 가입 신청
  - 승인 후 입장
  - 멤버 앱 홈/모듈 진입
- 아직 없는 것:
  - 클럽 전용 공지/이벤트/자료/갤러리 CRUD
  - 클럽 멤버 관리
  - 역할 변경
  - 초대 링크 발급/수락

### 초대제 클럽

- `invite-first` 정책은 있습니다.
- 하지만 실제 초대 토큰/링크/만료 처리는 아직 없습니다.

### DB 기본 전환

- 현재 기본은 로컬 파일 저장 흐름입니다.
- DB 기준 런타임은 준비돼 있지만 기본 모드는 아닙니다.

## 이번 변경의 핵심 의도

- 사용자는 자기 홈 안에서 대부분 시간을 보내야 합니다.
- 콘솔은 자주 들어가는 메인이 아니라, 가끔 여는 운영실이어야 합니다.
- 같은 카드 시스템을 써도 `공개 / 회원 / 관리자`가 한눈에 다르게 읽히게 만드는 것이 목표입니다.
