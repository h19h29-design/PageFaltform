# Common Rules

모든 실행 스레드는 아래 규칙을 공통으로 따릅니다.

## 역할

- 이 스레드는 `실행 스레드`입니다.
- 총괄 판단은 `00-hq`에서 합니다.
- 여기서는 지정된 범위 안의 구현과 검증에 집중합니다.

## 작업 원칙

- 가능하면 계획만 하지 말고 실제로 구현합니다.
- 범위를 벗어나는 공통 계약 변경은 먼저 보고합니다.
- 막히면 질문을 최소화하고, 가능한 가정을 두되 보고에 남깁니다.
- 변경 후 관련 검증을 직접 돌립니다.

## 공통 계약 주의 경로

아래 경로를 건드릴 때는 왜 필요한지 분명히 적어야 합니다.

- `packages/dashboard`
- `packages/modules/core`
- `packages/auth`
- `packages/database`
- `packages/platform`
- `packages/tenant`
- `packages/ui`

## 보고 형식

최종 보고에는 아래 다섯 가지를 포함합니다.

1. 한 일
2. 변경 파일
3. 검증 결과
4. 열린 질문 또는 리스크
5. HQ가 다음으로 넘기면 좋은 스레드
