# 04 Auth and Data

목표:

- 현재 데모/파일 기반 흐름을 실제 DB 중심 구조로 옮길 준비를 합니다.

범위:

- `packages/auth`
- `packages/database`
- `apps/web/app/(family)`
- `apps/web/app/(console)`

핵심 할 일:

- 가족 입장과 관리자 로그인 역할 분리 고정
- 역할 매트릭스 초안 정리
- DB 전환 시 필요한 세션과 스키마 요구사항 정리

하지 말 것:

- 홈 카드 계약 재정의
- 개별 모듈 카드 상세 구현

완료 기준:

- 어떤 데이터를 DB로 옮겨야 하는지 설명 가능함
- 어떤 인증 흐름이 가족용이고 관리자인지 더 분명해짐
