# tests

통합 테스트, E2E 테스트, 핵심 시나리오 테스트를 둡니다.

## 우선 자동화할 시나리오

- 관리자 콘솔 로그인 -> 가족 목록 조회
- 새 미니 가족 홈 생성 -> `/f/{slug}` -> `/app/{slug}` 연결 확인
- 빌더에서 모듈 순서/프리셋 변경 -> 저장 -> 홈과 입구 반영 확인
- 빌더 리셋 -> 기본 구성 복귀 확인
- `/api/health` 응답과 내부망 경로 접근 확인

## DB 전환 전에 꼭 필요한 회귀 묶음

- 파일 저장과 DB 저장이 같은 가족 프리뷰를 반환하는지 비교
- 파일 저장과 DB 저장이 같은 워크스페이스 draft를 반환하는지 비교
- 중복 슬러그, 빈 필수값, 모듈 비활성화 조합의 예외 처리 비교
- 저장 후 다른 기기에서 같은 결과를 읽는 시나리오 스모크 테스트

특히 "가족 진입 -> 메인 홈 -> 모듈 작성 -> 메인 노출" 흐름은 반드시 자동화 대상입니다.
