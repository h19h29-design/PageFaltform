# 공통 프로젝트 컨텍스트

## 프로젝트명

YSplan

## 현재 목표

가족별로 커스터마이징 가능한 멀티테넌트 가족 플랫폼을 만든다.

핵심 UX:

- 처음 화면은 예쁜 스플래시
- `들어가기` 버튼 클릭
- 비밀번호 또는 접근 코드 입력
- 가족 메인 홈 진입
- 각 게시판에서 체크한 항목이 메인 홈에 카드처럼 노출

## 현재 설정

- 아직 실제 앱 구현 전 단계
- 기획/아키텍처/로드맵/초안 문서는 이미 정리됨
- 권장 기술 방향은 `Next.js App Router + PostgreSQL + Prisma + Auth.js + monorepo`

## 현재 고정된 원칙

- 이 프로젝트는 장기적으로 여러 가족이 쓸 수 있는 플랫폼이어야 한다.
- 처음부터 모든 모듈을 다 만들지 않는다.
- 핵심 MVP부터 만든 뒤 확장 모듈을 붙인다.
- 공지 게시판은 초기부터 꼭 넣는다.
- 진행률/습관 모듈은 차별화 포인트로 본다.
- 모든 모듈은 공통 메타데이터 계약을 공유해야 한다.

## 현재 중요 모듈

### MVP 1차

- 공지 게시판
- 일반 글 게시판
- 갤러리
- 일정/캘린더
- 투두
- 메인 홈
- 가족 진입 화면

### MVP 2차

- 일기
- 학교/학원 시간표
- 24시간 하루 시간표
- 진행률 보드
- 습관 체크

## 역할 공통 지침

- 자기 역할 범위 안의 구조를 먼저 설계하거나 구현한다.
- 다른 역할과 겹치는 계약 변경은 문서에 남긴다.
- 시각 스타일, 세부 필드, 실제 사용자 흐름은 사용자와 추가 대화로 다듬는다.
- 너무 많은 기능을 한 번에 밀어 넣지 말고, 확장 가능한 구조를 먼저 만든다.
