# Control Tower

이 문서는 `현재 대화 스레드`를 총괄 허브로 운영하기 위한 기준서입니다.

목표는 하나입니다. 이 스레드에서 우선순위와 범위를 잡고, 다른 스레드에는 딱 잘린 실행 패킷만 보내서 병렬 속도를 높이는 것입니다.

## 역할 분리

### 총괄 스레드

- 우선순위 결정
- 스레드 생성 여부 판단
- 범위 분리
- 충돌 가능성 점검
- 공통 계약 승인
- 결과 취합과 통합 검토

### 실행 스레드

- 지정된 범위 안에서 구현
- 테스트와 검증 수행
- 변경 사항 보고
- 리스크와 열린 질문 반환

## 이 스레드에서 사용할 운영 명령

아래처럼 짧게 지시하면 됩니다.

- `03 스레드 열어`
- `03에 홈 카드 계약부터 시켜`
- `04와 05 병렬 패킷 줘`
- `02는 인증 DB 전환 범위만 잘라`
- `06은 내부망 운영과 배포 체크리스트만 먼저`
- `현황판 업데이트해`
- `통합 관점에서 리뷰해`

## 총괄 스레드의 기본 출력 형식

다른 스레드로 보낼 때는 아래 항목을 유지합니다.

1. 목표
2. 범위
3. 이번 스레드에서 하지 말 것
4. 먼저 읽을 문서
5. 수정 가능 경로
6. 완료 기준
7. 보고 형식

## 스레드로 넘기기 전에 확인할 것

- 이미 다른 스레드가 같은 파일이나 계약을 만지고 있는지
- 공통 패키지 변경이 필요한지
- 더 앞선 스레드의 산출물이 필요한지
- 문서로 먼저 고정해야 하는 결정인지

## 공통 계약 승인 대상

아래 경로는 가능하면 총괄 스레드에서 먼저 방향을 고정합니다.

- `packages/modules/core`
- `packages/dashboard`
- `packages/auth`
- `packages/tenant`
- `packages/database`
- `packages/ui`
- `docs/architecture.md`

## 현재 프로젝트 기준 운영 원칙

- `01 Foundation`은 기반선을 안정적으로 유지합니다.
- `03 Dashboard Home`은 홈 카드 계약의 기준선 역할을 합니다.
- `04`, `05`, `07`은 가급적 `03`의 카드 계약을 따라갑니다.
- `02 Entry and Auth`는 가족 입장과 관리자 인증을 계속 분리합니다.
- `06 Platform and Deploy`는 운영자 편의 기능과 저장/배포 체계를 담당합니다.

## 보고를 받으면 이 스레드에서 할 일

1. 결과를 요약합니다.
2. 다음 스레드에 필요한 결정을 분리합니다.
3. `THREAD-BOARD.md`를 갱신합니다.
4. 필요하면 후속 디스패치 패킷을 바로 만듭니다.
