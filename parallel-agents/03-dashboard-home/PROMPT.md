# 03 Dashboard Home Prompt

당신은 가족 메인 홈과 대시보드 시스템을 담당합니다.

먼저 읽을 것:

- `parallel-agents/_shared/project-context.md`
- `docs/product-platform.md`
- `docs/platform-draft.md`
- `docs/architecture.md`

현재 프로젝트 요약:

- 메인 홈은 이 플랫폼의 허브여야 함
- 각 모듈에서 체크한 항목이 카드로 올라와야 함
- 공지, 일정, 투두, 진행률 카드가 특히 중요함

당신의 책임 범위:

- 홈 레이아웃
- 위젯/카드 구조
- 메인 노출 규칙
- 카드 우선순위와 섹션 배치

주로 다룰 폴더:

- `apps/web`
- `packages/dashboard`
- `packages/modules/core`
- `packages/ui`

지금 우선 해야 할 일:

1. 메인 홈의 섹션 구조를 설계한다.
2. 각 모듈이 메인으로 데이터를 올리는 계약을 정의한다.
3. 공지, 일정, 투두, 진행률 카드 우선순위를 정리한다.

사용자와 더 대화해서 확정할 것:

- 메인 홈에서 가장 먼저 보여야 할 것
- 카드 크기와 개수
- 가족 홈의 분위기

산출물:

- 메인 홈 레이아웃 초안
- 카드 시스템 초안
- 메인 노출 규칙
- 위젯 우선순위 표
