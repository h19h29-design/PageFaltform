# 05 Schedule Modules Prompt

먼저 읽을 것:

- `parallel-agents/_shared/project-context.md`
- `docs/product-platform.md`
- `docs/platform-draft.md`
- `docs/architecture.md`

현재 프로젝트 요약:

- 일정, 투두, 학교/학원 시간표, 24시간 하루 시간표가 중요 모듈임
- 가족 메인 홈에서 오늘 기준 정보를 잘 뽑아야 함

당신의 책임 범위:

- 일정/캘린더
- 투두
- 학교/학원 시간표
- 24시간 하루 시간표

주로 다룰 폴더:

- `packages/modules/calendar`
- `packages/modules/todo`
- `packages/modules/school-timetable`
- `packages/modules/day-planner`
- `packages/modules/core`

지금 우선 해야 할 일:

1. 오늘 기준으로 메인에 무엇을 올릴지 정리한다.
2. 시간표와 일정의 차이를 분명히 나눈다.
3. 투두와 하루 시간표의 연결 가능성을 검토한다.

사용자와 더 대화해서 확정할 것:

- 시간표를 개인별로 나눌지
- 가족 공용 일정과 개인 일정의 비율
- 투두를 단순형으로 갈지, 담당자 기반으로 갈지

산출물:

- 시간형 모듈 설계안
- 메인 카드 집계 기준
- 오늘 화면 기준 정의
- 공통 시간 데이터 규칙
